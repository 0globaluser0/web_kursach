# Требования к блок-схемам

Правила основаны на материалах по ГОСТ 19.701.

## Общие правила

- Символ должен графически обозначать функцию, которую он выполняет.
- Символы должны быть расположены равномерно.
- Нужно избегать слишком длинных линий.
- Нужно минимизировать пересечения линий.
- Символы желательно делать одного размера.
- Предпочтительна горизонтальная ориентация символов.
- Текст внутри символов должен читаться слева направо и сверху вниз.
- Внутри символа размещается минимальный текст, достаточный для понимания действия.

## Соединения

- Потоки данных или управления показываются линиями.
- Стандартное направление потока: слева направо и сверху вниз.
- Если направление отличается от стандартного, ставятся стрелки.
- Пересечения линий лучше избегать.
- Пересекающиеся линии не означают логической связи.
- Линии должны входить в символ слева или сверху.
- Линии должны выходить из символа справа или снизу.
- Линии должны быть направлены к центру символа.

## Несколько выходов

Несколько выходов из символа можно показывать:

- несколькими линиями от одного символа;
- одной линией, которая затем разветвляется.

Каждый выход должен иметь условие, например:

- `Да`;
- `Нет`;
- `Данные корректны`;
- `Недостаточно средств`.

## Основные символы

| Символ | Назначение |
| --- | --- |
| Терминатор | начало или конец алгоритма |
| Процесс | выполнение действия |
| Решение | выбор ветви по условию |
| Ввод-вывод | ввод данных или вывод результата |
| Документ | документ или отчет |
| Соединитель | продолжение схемы |
| Комментарий | пояснение к элементу |

## Для текущего проекта

В блок-схеме нужно показать:

1. Открытие сайта.
2. Проверку авторизации.
3. Вход или регистрацию.
4. Проверку корректности данных.
5. Переход в личный кабинет.
6. Просмотр рынка.
7. Выбор актива.
8. Ввод количества.
9. Покупку или продажу.
10. Проверку достаточности средств или монет.
11. Обновление баланса.
12. Создание заявки.
13. Создание транзакции.
14. Отображение портфеля и истории.

Текущая схема находится в `PZ/block_scheme.mmd`.
