# Приложения, список источников и список сокращений

## Список сокращений

Список сокращений должен идти перед списком источников.

Локальное требование: желательно сделать объемом около половины страницы.

Для текущего проекта можно использовать:

| Сокращение | Расшифровка |
| --- | --- |
| БД | база данных |
| ПО | программное обеспечение |
| ПЗ | пояснительная записка |
| ИС | информационная система |
| UI | пользовательский интерфейс |
| UX | пользовательский опыт |
| HTML | язык гипертекстовой разметки |
| CSS | каскадные таблицы стилей |
| JS | JavaScript |
| DOM | объектная модель документа |
| API | программный интерфейс приложения |
| CRUD | создание, чтение, изменение и удаление данных |
| USD | доллар США |
| BTC | Bitcoin |
| ETH | Ethereum |
| SOL | Solana |
| ADA | Cardano |
| localStorage | локальное хранилище браузера |

## Список источников

Список источников должен быть нумерованным.

Сортировка допускается:

- по алфавиту первого слова;
- или по порядку появления ссылок в тексте.

Ссылки в тексте оформляются в квадратных скобках:

`Для хранения данных используется localStorage [3].`

Для интернет-источников указываются:

- название материала;
- организация или сайт;
- URL;
- дата обращения.

Пример:

`MDN Web Docs. Window: localStorage property [Электронный ресурс]. URL: https://developer.mozilla.org/ (дата обращения: 11.06.2026).`

## Приложения

Приложения обозначаются буквами русского алфавита.

Не используются буквы:

- Ё;
- З;
- Й;
- О;
- Ч;
- Ь;
- Ы;
- Ъ.

Пример:

- `ПРИЛОЖЕНИЕ А`;
- `ПРИЛОЖЕНИЕ Б`;
- `ПРИЛОЖЕНИЕ В`.

## Документированные приложения

Документированное приложение - отдельный программный документ по ГОСТ 19.101-87.

Структура такого приложения:

1. Титульный лист.
2. Аннотация документа.
3. Содержание.
4. Основные разделы.

Для текущего курсового достаточно оформить приложение с текстом программы и описанием файлов, если преподаватель не требует полноценный отдельный программный документ.

## Иллюстрирующие приложения

К иллюстрирующим приложениям относятся материалы, не выделенные ГОСТом как отдельные программные документы:

- скриншоты экранных форм;
- схема базы данных;
- дополнительные таблицы;
- диаграммы;
- блок-схемы;
- вспомогательные рисунки.
