# Требования к курсовому проекту

## Реальная часть

Сайт должен содержать:

- авторизацию;
- регистрацию;
- не менее пяти связанных таблиц/сущностей базы данных;
- торговую или предметную логику, связанную с выбранной темой;
- рабочую навигацию между страницами;
- пользовательскую часть;
- административную часть, если она нужна по логике проекта.

В текущем проекте тема - учебная симуляция торговли криптовалютой.

## Требования к данным

В проекте должны быть пять связанных таблиц базы данных. В текущей реализации описаны семь связанных сущностей:

| Сущность | Назначение |
| --- | --- |
| `roles` | Роли пользователей |
| `users` | Пользователи, учетные данные, USD-баланс |
| `currencies` | Криптовалюты, цена, описание, риск |
| `wallets` | Балансы пользователей по криптовалютам |
| `orders` | Торговые заявки |
| `transactions` | Завершенные операции |
| `price_history` | История изменения курсов |

Данные сайта фактически хранятся в `localStorage`, но для ПЗ описываются как логическая модель базы данных.

## Обязательные материалы для отчета

По `требования.txt` в отчете должны быть:

- блок-схема;
- алгоритмы работы программы;
- схема базы данных;
- анализ предметной области;
- раздел о базах данных;
- описание разработки сайта;
- тестирование;
- заключение;
- список сокращений;
- список источников;
- приложение А с описанием каждого файла и кодом;
- приложение Б с блок-схемой;
- приложение В со скриншотами, рисунками или кодом базы данных по желанию.

## Объем

Локальное требование преподавателя: 30-40 страниц.

Методичка по ПЗ указывает общий объем 40-80 листов без документированных приложений. Для этого курсового проекта ориентиром считать 30-40 страниц, потому что это явно указано в локальных требованиях.

## Важные замечания преподавателя

- Шрифт основного текста: 13.
- Подглавы/заголовки: отступ после заголовка 39 пт.
- Пустого места на странице не должно быть больше 1/4 страницы.
- Рисунки должны быть центрированы.
- Подпись к рисунку находится снизу.
- Подпись к таблице находится сверху.
- При переносе таблицы писать `Продолжение таблицы X.Y` и повторять шапку.
