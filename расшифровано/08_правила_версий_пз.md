# Правила версионирования Word-файла ПЗ

## Главное правило

При любой правке, доработке, добавлении раздела, таблицы, рисунка, приложения или любого другого пункта в Word-документ ПЗ необходимо создавать новый `.docx`-файл.

Предыдущие версии не перезаписывать и не удалять.

## Формат имени

Имя файла должно быть коротким и начинаться с `ПЗ`.

Формат:

```text
ПЗ_001_YYYY-MM-DD_HH-MM.docx
```

Пример:

```text
ПЗ_001_2026-06-11_11-12.docx
ПЗ_002_2026-06-11_11-35.docx
ПЗ_003_2026-06-12_09-10.docx
```

## Значение меток

| Часть имени | Значение |
| --- | --- |
| `ПЗ` | пояснительная записка |
| `001` | порядковый номер версии |
| `YYYY-MM-DD` | дата создания версии |
| `HH-MM` | время создания версии |

## Как выбирать номер

1. Посмотреть существующие файлы вида `ПЗ_*.docx` в папке `PZ`.
2. Найти самый большой номер.
3. Создать новый файл со следующим номером.

Если файлов еще нет, первая версия:

```text
ПЗ_001_YYYY-MM-DD_HH-MM.docx
```

## Генератор

Файл `PZ/build_pz_docx.py` настроен автоматически создавать новый файл по этой схеме.

Не нужно вручную перезаписывать старые файлы.

## Требование по шрифту

По локальным требованиям и методичке для этой ПЗ используется:

- основной текст: `Times New Roman 13`;
- код в приложении: `Courier New 9`.

Если преподаватель отдельно потребует `Times New Roman 14`, нужно изменить генератор и создать новую версию ПЗ по тем же правилам.
