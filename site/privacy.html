Политика конфиденциальности CryptoTrade

Редакция от 11.06.2026. Документ предназначен для учебного проекта и описывает обработку демо-данных в локальной базе SQLite.

1. Общие положения

CryptoTrade является учебным сайтом для демонстрации регистрации, авторизации, просмотра рынка, торговли и администрирования. Сайт не является реальной биржей, брокером, платежным сервисом или инвестиционной платформой.

2. Какие данные используются

В проекте используются имя пользователя, email, пароль для учебного входа, демо-баланс, состав портфеля и история операций. Эти сведения нужны только для показа работы интерфейса и логики сайта.

3. Где хранятся данные

Данные сохраняются в локальном файле SQLite cryptotrade.db, который создается рядом с сайтом при запуске backend.py. Проект не отправляет учетные записи, портфель и историю операций на внешний сервер разработчика.

4. Внешние курсы

Для автоматического обновления цен сайт может обращаться к открытому рыночному API Binance Spot по публичным торговым парам, например BTCUSDT или ETHUSDT. При таком запросе внешний сервис получает технические сведения браузера, необходимые для ответа на HTTP-запрос.

5. Ограничения безопасности

Пароли в учебной версии хранятся в локальной базе без хеширования, поэтому в проекте нельзя использовать реальные пароли, персональные данные или финансовую информацию. Демо-аккаунты и демо-балансы не имеют денежной ценности.

6. Очистка данных

Данные можно удалить, остановив сервер и удалив файл site/cryptotrade.db. После повторного запуска сервер создаст базу заново, а сайт заполнит начальные демонстрационные данные.

7. Изменения политики

При доработке проекта политика может обновляться вместе с функциональностью сайта. Актуальная версия документа хранится в файле privacy.html.
