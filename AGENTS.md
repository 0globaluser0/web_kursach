## Основные требования преподавателя

- Сайт и ПЗ являются двумя частями курсового проекта.
- В сайте должна быть авторизация и регистрация.
- Нужно не менее пяти связанных таблиц или сущностей БД.
- В ПЗ должны быть анализ предметной области, база данных, разработка, тестирование, заключение, список сокращений, источники и приложения.
- Нужно добавить блок-схему алгоритма работы программы.
- Нужно добавить схему базы данных.
- Примерный объем ПЗ: 30-40 страниц.
- Основной шрифт ПЗ: Times New Roman 13.
- Код в приложении: Courier New 9.
- Подписи рисунков снизу, таблиц сверху.
- Пустого места на странице не должно быть больше 1/4 страницы.

## Правило версий Word-файла ПЗ

- При любой правке, доработке или добавлении пункта в Word-документ ПЗ нужно создавать новый `.docx`, а не перезаписывать предыдущий.
- Имя файла должно начинаться с `ПЗ`.
- Формат имени: `ПЗ_001_YYYY-MM-DD_HH-MM.docx`, где `001` - порядковый номер версии, а дата и время - момент создания файла.
- Следующая версия должна увеличивать номер: `ПЗ_002_...`, `ПЗ_003_...` и далее.
- Старые файлы ПЗ не удалять: они нужны как история версий.
- Генератор `PZ/build_pz_docx.py` уже настроен создавать файлы по этой схеме.

## Предпочтительный порядок будущей работы

1. Сначала читать `AGENTS.md`.
2. Затем смотреть `расшифровано/07_чеклист_для_текущей_пз.md`.
3. Для оформления использовать `расшифровано/03_оформление_по_гост.md`.
4. Для рисунков, таблиц и кода использовать `расшифровано/04_рисунки_таблицы_формулы_код.md`.
5. Для структуры ПЗ использовать `расшифровано/02_структура_пояснительной_записки.md`.
6. При изменении сайта сверяться с `site/assets/js/app.js` и страницами `site/*.html`.
7. При изменении Word-файла ПЗ соблюдать правило версий: новый файл `ПЗ_XXX_YYYY-MM-DD_HH-MM.docx`.
